Fingi di essere un esperto di c++ e analisi di analisi statica.
Analizza il file `doc/output-analisi/analisi.log` e crea un riassunto strutturato degli errori di analisi statica.

**IMPORTANTE - Cosa fare**:
1. LEGGI il file analisi.log usando il tool `view`
2. FILTRA solo gli errori dei file del progetto (main.cpp, engine/*.cpp, piece/*.hpp, board/*.cpp, etc.)
3. ESCLUDI completamente:
  - `script/*/**`
  - `stockfish/**`
  - Qualsiasi codice di terze parti
4. RAGGRUPPA errori identici contando le occorrenze
5. CREA o SOVRASCRIVI il file `doc/testi-llm/riassunto-analisi-statica.md`

**Struttura del documento da creare**:

```markdown
# Riassunto Analisi Statica - Chess Engine C++23

## Sommario Esecutivo
- File analizzati: [numero]
- Errori totali: [numero per categoria]

## Errori Raggruppati

### [Nome Errore]
**Fonte**: CPPCHECK/clang-tidy/...
**Tipo**: style/warning/error
**Descrizione**: [se disponibile o necessaria]
**Occorrenze**: [numero]
**File**: [lista file:riga]
**Fix suggerito**: [esempio codice]
**Miglioramenti nel progetto se applicata**: [stima performance]

[Ripeti per ogni tipo di errore]

## Priorità
- Alta: [errori critici]
- Media: [warning importanti]
- Bassa: [style issues]

Output richiesto:

- Tutto in italiano
- Usa tabelle markdown per leggibilità
- Fornisci esempi di fix concreti
```
